## Work Summary

Diagnosed and fixed semantic search 502 error caused by URL configuration sanitization stripping trailing slash from FastAPI endpoint.

## Root Cause

`embeddings_url` in `solr-search/config.py` was applying `.rstrip("/")`, breaking the endpoint URL:
- Expected: `http://embeddings-server:8001/v1/embeddings/`
- After sanitization: `http://embeddings-server:8001/v1/embeddings` (no slash)
- FastAPI route `@app.post("/v1/embeddings/")` did not match → 502

## Decision

**Do not strip trailing slashes from complete endpoint URLs.** URL sanitization is appropriate for base URLs used in concatenation, but not for finished endpoints passed to HTTP clients.

## Changes Made

- **File:** `src/solr-search/config.py` line 90
- **Change:** Removed `.rstrip("/")` from embeddings_url default
