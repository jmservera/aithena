## Summary

Added `intel-extension-for-pytorch` (IPEX) to the OpenVINO optional-dependencies group in `src/embeddings-server/pyproject.toml`. Regenerated `uv.lock`. This bridges the gap between PyTorch's Intel XPU hardware detection and Intel's XPU runtime — without IPEX, inference fails even though hardware is detected.

## Changes

- `src/embeddings-server/pyproject.toml` — Added IPEX to `[project.optional-dependencies] openvino`
- `uv.lock` — Regenerated with IPEX 2.8.0 (resolves cleanly with torch 2.10.0)

## Impact

- OpenVINO container images now have full Intel GPU/XPU support
- CPU-only builds are unaffected (IPEX only in openvino extras)
- Existing `docker/compose.gpu-intel.yml` continues to work without changes

## Architecture Notes

Dependency chain: `uv sync --extra openvino` → includes IPEX 2.8.0 automatically. No application code changes required; existing conditional `import openvino` logic detects and uses it.
