## Summary

Implemented the OQ-1 fanout exchange pattern for dual-model A/B testing. Both `document-indexer` and `document-indexer-e5` now receive every document via a shared fanout exchange.

## Changes

### document-lister
- Added `EXCHANGE_NAME` config variable (default: `documents`)
- Changed `channel.basic_publish()` from direct queue (`exchange=""`) to fanout exchange (`exchange=EXCHANGE_NAME`)
- Routing key changed from `QUEUE_NAME` to `""` (fanout ignores routing key)
- Idempotent exchange declaration at startup

### document-indexer
- Added `EXCHANGE_NAME` config variable (default: `documents`)
- Added queue-to-exchange binding in `get_queue()`: `channel.queue_bind(queue=QUEUE_NAME, exchange=EXCHANGE_NAME)`
- Idempotent exchange + binding declarations on every reconnect
- Backward compatible: unchanged message format, callback logic, Redis key isolation

## Backward Compatibility

- No changes to Redis tracking (already parameterized by `QUEUE_NAME`)
