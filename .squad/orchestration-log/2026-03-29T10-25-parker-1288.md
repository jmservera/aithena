# Parker: Issue #1288 — Extract Shared Auth Library (aithena-common)

**Timestamp:** 2026-03-29T10:25:00Z  
**Status:** ✅ Complete  
**Tests:** All tests pass

## Summary

Extracted shared auth library into `src/aithena-common/` to break the dependency between installer and solr-search. Removed sys.path manipulation hack. Both installer and solr-search now depend cleanly on aithena-common as a shared package.

## Changes

1. **Created src/aithena-common/** — New shared Python package
   - `aithena_common/passwords.py` — `hash_password()`, `verify_password()`, `check_needs_rehash()`
   - `aithena_common/auth_db.py` — `init_auth_db()`, `find_user()`, `get_schema_version()`, `SCHEMA_VERSION`

2. **Updated installer/** — Now a proper uv project
   - Added `pyproject.toml` with `aithena-common` as path dependency
   - Removed `sys.path` manipulation
   - `ensure_runtime_dependencies()` fallback uses `--project installer` instead of `--project src/solr-search`

3. **Updated src/solr-search/**
   - Added `aithena-common` to dependencies
   - Auth module imports from `aithena_common` instead of local duplication
   - `reset_password.py` re-exports from local `auth` module for backward compatibility

## Impact

- Installer is now a proper, installable package (not a sys.path hack)
- Future services needing auth DB access should depend on `aithena-common`, not import from solr-search
- Follows Dependency Inversion Principle — all services depend on shared entities, not on each other's internals
- Foundation for Phase 2: expand aithena-common with shared auth models, JWT utils, logging

## What Stays in solr-search

- JWT creation/verification and token handling
- User CRUD (create_user, list_users, update_user, delete_user)
- Migrations framework and default admin seeding
- Password policy validation
- All HTTP/FastAPI-specific auth logic

## Architecture

```
installer ──\
solr-search ─┼──> aithena-common (passwords, auth_db)
other ────/
```

No service imports from another service's implementation. All shared logic flows through aithena-common.
