## Summary

Collection parameter API implementation for routing search queries to different Solr collections during the A/B test. All CI checks passed; no review comments.

## Implementation Details

- **Optional `collection` query param** on `/v1/search/`, `/v1/facets/`, `/v1/books/` endpoints
- **Allowlist validation** against `ALLOWED_COLLECTIONS` env var (default: `"books"`)
- **Env var config:**
  - `ALLOWED_COLLECTIONS` — comma-separated list of allowed collections
  - `DEFAULT_COLLECTION` — used when param omitted (default: `"books"`)
  - `E5_COLLECTIONS` — collections requiring `input_type="query"` for embeddings
  - `EMBEDDINGS_URL_{UPPER_NAME}` — per-collection embeddings server URL override
- **Backward compatible** — keyword-only `collection` param; existing callers unaffected

## Impact

- `solr-search` can now route queries to `books` or `books_e5base` based on runtime config
- Frontend (Dallas) can add A/B UI toggle with `collection` query param when ready
- Infrastructure (Brett) can configure A/B test environment with per-collection embeddings servers
- Ready for Phase 2 indexing and benchmarking
