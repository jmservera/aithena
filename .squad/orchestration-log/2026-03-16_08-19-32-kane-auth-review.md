## Blockers Identified & Fixed

### 1. Hardcoded JWT Secret Fallback
**Severity:** 🔴 CRITICAL  
**Finding:** JWT secret had a hardcoded fallback if `JWT_SECRET` env var was missing.  
**Fix:** Environment variable is now mandatory; raises `ConfigurationError` if missing or empty.  
**Verification:** Tested with missing env; confirms exception.

### 2. JWT Expiration Not Enforced
**Severity:** 🔴 CRITICAL  
**Finding:** Token validation did not check `exp` claim; expired tokens still accepted.  
**Fix:** Added explicit expiration timestamp check in JWT decode pipeline.  
**Verification:** Token after expiry now rejects with `TokenExpiredError`.

### 3. No Login Rate Limiting
**Severity:** 🟠 HIGH  
**Finding:** `/auth/login` endpoint accepts unlimited failed attempts; brute-force vector.  
**Fix:** Implemented Redis-backed rate limiter (10 attempts / 15 minutes per IP).  
**Verification:** Manual test: 11th attempt in window triggers 429 response.

## Additional Review Notes

- Argon2id parameters acceptable for single-node deployment (work_factor=2, sensible for 128-256MB instances)
- Password reset flow requires explicit token redemption (no auto-login) ✅
- PII scrubbing on auth logs (no email in error messages) ✅
- SQL injection prevention via parameterized queries ✅
