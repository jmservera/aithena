### 2. Dependabot Automerge Trigger Bug (PR #1415)

**Problem:** `dependabot-automerge.yml` workflow never triggered despite successful CI runs on Dependabot PRs.

**Root Cause:** `branches: [dev]` filter on `workflow_run` trigger only matches the **source branch** of the CI run, not the PR base branch. Dependabot creates branches like `dependabot/pip/...`, which never match `[dev]`.

**Solution:** Remove `branches` filter entirely. Three layers of safety gates make it redundant:
- `gh pr list --head $HEAD_BRANCH` naturally finds PRs targeting dev
- Author check: `author.login == "app/dependabot"`
- Semver gating: patch/minor only

**Implementation:**
```yaml
on:
  workflow_run:
    workflows: ["CI - Unit & Integration Tests"]
    types: [completed]
```

**Impact:** Automerge will now work for patch/minor dependabot PRs.

**Related Decision:** `.squad/decisions.md` — "Remove `branches` Filter from Dependabot Automerge Workflow"
