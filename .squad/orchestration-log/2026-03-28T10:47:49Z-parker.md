### Issue: Similar Books Not Working in Semantic Search Mode (#1260)

**Problem:** When users search semantically and click "similar books" from chunk results, the endpoint received a chunk ID (e.g., `abc123_chunk_0000`) instead of the parent book ID, causing lookup failures.

**Solution Implemented:**

1. **Backend Resilience:** Modified `/v1/books/{document_id}/similar` endpoint in `src/solr-search/main.py` to detect chunk IDs and automatically resolve them to parent book IDs.
   - Check for `"_chunk_"` substring in ID
   - Extract parent ID (part before `_chunk_`)
   - Proceed with similarity search using parent ID

2. **Data Enrichment:** Updated `normalize_book()` in `src/solr-search/search_service.py` to return both:
   - `chunk_id` — the document ID from Solr (for semantic search results)
   - `parent_id` — the parent book ID (for the similar books endpoint)

3. **Test Coverage:** Updated `src/solr-search/tests/test_search_service.py` to verify the new `parent_id` field is correctly populated.
