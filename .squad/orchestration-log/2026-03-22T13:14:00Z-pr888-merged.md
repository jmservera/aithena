## Summary

Implemented standardized benchmark query suite for A/B testing distiluse (512D) vs e5-base (768D) embedding models.

### Deliverables

- **`scripts/benchmark/queries.json`** — 30 queries across 5 categories
  - Simple keyword (5) — baseline catalog searches
  - Natural language (6) — semantic search focus
  - Multilingual (6) — Spanish, Catalan, French
  - Long/complex (4) — e5-base context window advantage
  - Edge cases (9) — robustness testing (single chars, stopwords, accents, nonsense)

- **`scripts/benchmark/run_benchmark.py`** — Query runner with Jaccard overlap metric
  - Hits `GET /search` against live instance with collection parameter
  - Compares top-10 results across baseline and candidate collections
  - Jaccard similarity = |intersection| / |union|
  - Flags low-overlap queries (< 0.3) for manual review
  - JSON output for dashboard integration

### Design Decisions

- **No `input_type` injection in runner** — solr-search API handles `input_type=query` internally; runner just passes collection name
- **Jaccard metric** — Simple, interpretable overlap without requiring ground-truth relevance labels (no nDCG/MAP yet)
- **Top-10 evaluation** — Sufficient for human evaluation and representative of typical user experience

## Impact

- Phase 2-2 infrastructure complete; Phase 2 evaluation can proceed
- Benchmark scores reproducible and comparable across model iterations
- 30-query suite human-reviewable and extensible for future categories
