# Aithena v1.8.2 Release Notes — Infrastructure & Admin Improvements

_Date:_ 2026-03-19  
_Prepared by:_ Newt (Product Manager)

Aithena v1.8.2 is a maintenance release focusing on infrastructure improvements, file upload reliability, and modernization of the admin dashboard. This release removes the legacy Streamlit admin service, adds infrastructure management links to the React admin page, fixes file upload size handling, and improves CI/CD automation.

## Summary of shipped changes

### Infrastructure & Reliability

- **File upload size limit fix** — Fixed nginx reverse proxy configuration to allow uploads up to 64MB (previously rejecting files at the proxy layer). The 64MB limit gives headroom above the backend's 50MB maximum, preventing confusing rejection errors. Frontend error messaging improved to be generic rather than hardcoding the limit. (#596)
- **Dependabot CI automation fix** — Fixed the automerge workflow's environment detection. Previously, the workflow relied on file-based detection (`pyproject.toml`) to choose between `uv` and `pip`, but `embeddings-server` now includes `pyproject.toml` while still using `pip`. Changed to service-name-based detection for reliable CI behavior. (#598)

### Admin Dashboard Modernization

- **Removed Streamlit admin service** — The legacy Python Streamlit-based admin dashboard (`src/admin/`) has been removed from the deployment stack. The Docker Compose service, nginx proxy configuration, and backend monitoring have all been cleaned up. The React admin page (`/admin/`) now provides full feature parity, including document upload, queue monitoring, and system status. A 301 redirect from `/admin/streamlit/` to `/admin/` ensures graceful transition for existing bookmarks and scripts. (#587)

- **Infrastructure management links** — The React admin page now includes an Infrastructure section with card-style links to external admin tools: **Solr Admin** (`/admin/solr/`) for search index management and **RabbitMQ Management** (`/admin/rabbitmq/`) for queue monitoring. All labels and descriptions are fully localized in English, Spanish, Catalan, and French. (#588)

## Upgrade instructions

### For users running Aithena v1.8.1

1. Update to the v1.8.2 release:
   ```bash
   git fetch origin
   git checkout v1.8.2
   ```

2. Verify the fixes:
   - **Upload fix:** Upload a file larger than previously accepted (e.g., 30MB+) to confirm it no longer fails with a size error
   - **Admin dashboard:** Navigate to `/admin/` and confirm the Infrastructure section displays links to Solr Admin and RabbitMQ Management
   - **Streamlit redirect:** If you have bookmarks to `/admin/streamlit/`, verify they automatically redirect to `/admin/`

3. No new environment variables, configuration changes, or database migrations are required.

### For Docker Compose deployments

If deploying via Docker Compose:

```bash
docker compose pull
docker compose up -d
```

The following changes are automatically applied:

- Nginx proxy configuration updated to allow larger uploads
- `streamlit-admin` service is no longer deployed (safe to remove if present in your override file)
- React admin page now displays infrastructure links
- Dependabot CI workflows use service-name detection for Python dependency management

### Streamlit service removal

If you have customized docker-compose files or CI workflows that reference the `streamlit-admin` service:

- Remove the `streamlit-admin` service definition from `docker-compose.yml`
- Remove any environment variables prefixed with `STREAMLIT_`
- Remove nginx location blocks for `/admin/streamlit/` (the v1.8.2 compose includes a redirect for backward compatibility)
- All Streamlit-specific dependencies can be removed from the container build

The `src/admin/` source directory is preserved for reference and historical documentation but is not included in the Docker build.
