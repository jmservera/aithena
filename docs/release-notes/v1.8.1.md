# Aithena v1.8.1 Release Notes — Patch Release

_Date:_ 2026-03-19  
_Prepared by:_ Newt (Product Manager)

Aithena v1.8.1 is a patch release addressing critical bugs discovered after v1.8.0 release. This release fixes incomplete internationalization translations, corrects the Stats UI service status display, and resolves an admin page login loop that prevented admin access.

## Summary of shipped changes

### Bug Fixes

- **Incomplete i18n translations** — Resolved missing translations on Search, Library, and Upload pages, ensuring full multilingual support across all user-facing interfaces (#564)
- **Stats UI service status display** — Fixed Stats widget to correctly display all service statuses (Redis, RabbitMQ, Solr, embeddings-server) instead of incorrectly marking RabbitMQ as down (#563)
- **Admin page infinite login loop** — Fixed authentication flow that was preventing admin users from accessing the admin dashboard due to repeated login redirects (#561)
- **Version display** — Ensured the application version display correctly shows the actual VERSION file value in the interface (#569)

## Upgrade instructions

### For users running Aithena v1.8.0

1. Update to the v1.8.1 release:
   ```bash
   git fetch origin
   git checkout v1.8.1
   ```

2. Verify the fixes:
   - **i18n fix:** Switch language in settings; Search, Library, and Upload pages should display translations for all languages
   - **Stats fix:** Admin dashboard Stats widget should show all services (Redis ✓, RabbitMQ ✓, Solr ✓, embeddings-server ✓)
   - **Admin login fix:** Admin users should be able to access the admin dashboard without repeated login redirects
   - **Version display fix:** Application version display should show the correct release (v1.8.1)

3. No new environment variables, configuration changes, or database migrations are required.

### For Docker Compose deployments

If deploying via Docker Compose:

```bash
docker compose pull
docker compose up -d
```

All fixes are automatically active upon deployment.

## Validation checklist

Before using v1.8.1 in production, verify:

- [ ] Language switcher displays translations on Search, Library, and Upload pages
- [ ] Admin Stats widget shows all services as operational
- [ ] Admin user can log in without repeated redirects
- [ ] Application version display shows v1.8.1 (not v1.8.0 or stale version)
- [ ] No console errors in browser DevTools
- [ ] All services start cleanly: `docker compose up`
