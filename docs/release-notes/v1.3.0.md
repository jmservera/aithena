# Aithena v1.3.0 Release Notes — Backend Excellence & Observability

_Date:_ 2026-03-17  
_Prepared by:_ Newt (Product Manager)

Aithena **v1.3.0** is a major backend quality and operational observability release. It introduces structured JSON logging across all services, admin dashboard authentication, comprehensive test coverage instrumentation, URL-based search state management, resilient circuit breaker patterns, correlation ID tracking for request tracing, and extensive observability documentation. This release delivers enterprise-grade operational visibility and monitoring capabilities.

## Summary of shipped changes

### Backend Quality & Observability (BE-1 through BE-7)

- **Structured JSON logging** across all Python services (solr-search, document-indexer, document-lister, admin) provides consistent, machine-parseable log output for operational analysis (#336).
- **Admin dashboard authentication** protects the Streamlit admin interface with login page, JWT-based session management, and password hashing (#337).
- **pytest-cov coverage instrumentation** with HTML report generation in CI/CD enables visibility into test coverage metrics and progress toward ≥80% coverage target (#338).
- **URL-based search state management** using React Router's `useSearchParams` enables shareable search links with all filters, sort, and pagination preserved in the URL (#339).
- **Graceful degradation for Redis and Solr failures** via circuit breaker pattern prevents cascading failures and allows services to degrade gracefully instead of crashing (#340).
- **Correlation ID tracking** across service boundaries (HTTP, RabbitMQ, logs) enables end-to-end request tracing through the entire system (#341).
- **Observability runbook** documents log analysis workflows, request tracing procedures, and debugging strategies for operators (#342).

### Integration & Quality Assurance

- **Integration tests for auth and URL state** verify end-to-end behavior for admin dashboard authentication flow and URL-based search persistence (#343).

## Breaking changes

**Logging format change for operators:**

- All Python services now emit structured JSON logs instead of human-readable text. Tools and dashboards that parse logs must update to consume JSON format.
- Log level configuration moves from hardcoded level to environment variable `LOG_LEVEL` (default: `INFO`).
- Internal log messages no longer include stack traces by default; use `DEBUG` level or correlation ID lookup for detailed diagnostics.

**Admin dashboard access:**

- The Streamlit admin dashboard now requires authentication. Operators must log in with valid credentials before accessing the dashboard.
- Anonymous access to `/admin/streamlit/` is no longer permitted; requests redirect to `/login`.

**Search URL parameter structure:**

- Shareable search links now encode all filter state in URL query parameters (e.g., `?q=keyword&filters=year:2020,language:en&sort=relevance&page=2`).
- Old search URLs without query parameters may not restore full filter state; users should regenerate shareable links after upgrade.

## User-facing improvements

- **Shareable search links:** Copy the current search URL to share a filtered, sorted view with colleagues. Recipients receive the exact same results without re-running the search manually.
- **Browser back/forward:** Navigate search history using browser back/forward buttons; each page, sort, and filter change is tracked in the browser history.
- **Graceful degradation:** If Redis or Solr experience temporary issues, Aithena falls back gracefully instead of crashing. Status page shows degraded state with details.

## Backend improvements

- **Structured logging:** All Python services emit JSON logs with consistent schema, enabling automated parsing, alerting, and analysis tools.
- **Correlation IDs:** Each request carries a unique correlation ID through all services, enabling end-to-end request tracing in logs.
- **Circuit breaker resilience:** Redis and Solr timeouts no longer cascade failures; services continue operating at reduced capacity.
- **Admin security:** Admin dashboard protected by login; no anonymous access permitted.
- **Test coverage visibility:** HTML coverage reports generated in CI show per-file, per-function coverage metrics and help identify untested code paths.

## Security improvements

- **Admin dashboard authentication:** Login required for admin access; passwords hashed with bcrypt.
- **Session isolation:** Streamlit admin dashboard uses browser-session-based JWT tokens; no persistent credentials stored on client.
- **Correlation ID audit trail:** Every request, log message, and service-to-service call tagged with correlation ID for forensic investigation.

## Upgrade instructions

For users and operators moving to **v1.3.0**:

1. Update to the **v1.3.0** release commit or tag once published.
2. Redeploy all Python services (`solr-search`, `document-indexer`, `document-lister`, `admin`) to receive structured JSON logging.
3. Update log analysis tools and monitoring dashboards to consume JSON log format; see `docs/observability-runbook.md` for JSON parsing examples.
4. Configure log level if needed via `LOG_LEVEL` environment variable (default: `INFO`).
5. Re-run the installer to provision admin authentication if needed:
   ```bash
   python3 -m installer --reset
   ```
6. Redeploy `aithena-ui` to receive URL-based search state management.
7. No database migrations required; all changes are backward-compatible at the API level.
8. Run the integration test suite to validate end-to-end auth and search state:
   ```bash
   docker compose -f docker-compose.e2e.yml up --abort-on-container-exit
   ```

## Validation highlights

- **Structured logging:** All services consistently emit JSON logs with correlation ID, timestamp, level, service name, and message fields.
- **Admin auth:** Login flow validated with bcrypt password hashing; JWT tokens issued and validated correctly; logout clears session state.
- **Coverage visibility:** pytest-cov reports show coverage metrics for all services; target ≥80% test coverage achieved or flagged in CI.
- **URL state:** Search state persisted in URL query parameters; browser back/forward works correctly; shareable links restore exact filter/sort/page state.
- **Circuit breaker:** Timeouts on Redis or Solr connections trigger circuit breaker; fallback behavior prevents cascading failures; services remain operational in degraded mode.
- **Correlation IDs:** All requests tagged with UUID v4 correlation IDs; IDs propagated through RabbitMQ headers, HTTP response headers, and all log entries.
- **Observability runbook:** Comprehensive guide for operators on log analysis, request tracing, and debugging workflows.
- **Integration tests:** End-to-end tests verify admin auth flow and URL state persistence; all tests pass in CI.
