## Verified Scope for v1.16.0

### Search UI Bug Fixes (#1221 – #1224)

Four regressions in the search results display were resolved in this release:

| Issue | Description | Fix |
|---|---|---|
| #1224 | Semantic search chunk text rendered in a small, differently styled box | Chunk text now uses the same UI container and visual style as keyword results |
| #1223 | Keyword search context text was too long (no length limit) | Context text truncated to ~40 words for readability |
| #1222 | Page number/link missing from keyword search results (regression) | Keyword search results correctly show page numbers and allow opening the PDF at the match page |
| #1221 | Thumbnails missing from semantic and hybrid search results | Thumbnails now shown for all search modes (keyword, semantic, hybrid) |

These fixes restore and improve the search result display to a consistent, readable state across all search modes.

### Similar Books Fix (#1220)

The similar-books endpoint returned HTTP `422` in the v1.16 release candidate. Root cause: the kNN query was targeting parent book documents instead of chunk documents (where embeddings are stored). The fix redirects the kNN query to chunk documents, restoring the similar-books feature.

### Admin UI Improvement (#1138)

The admin indexing status list is now paginated. Previously the list was unbounded, which could degrade the admin dashboard with large book libraries. Pagination is applied automatically with no operator configuration needed.

### Infrastructure Fixes

| Issue | Area | Fix |
|---|---|---|
| #1137 | Thumbnails returned `text/html` instead of `image/jpeg` from nginx | nginx static content routing corrected for thumbnail paths |
| #1136 | RabbitMQ startup warning: `management_metrics_collection` feature deprecated | RabbitMQ configuration updated to suppress deprecation warning |
| #1132 | Admin container smoke test (release.yml) failing due to 30 s timeout | Smoke test timeout increased / health check approach updated |

### Pre-Release Container Workflow (New CI Feature, #1118 – #1123)

A formal release-candidate (RC) build and testing workflow was introduced to allow integration validation before official release tagging:

| Issue | Deliverable |
|---|---|
| #1120 | Reusable container build workflow (`.github/workflows/build-containers.yml`) covering all 6 services |
| #1123 | New `.github/workflows/pre-release.yml` — triggers on manual dispatch (dev branch) and on PRs to main |
| #1119 | RC auto-increment logic — queries ghcr.io for existing RC tags and increments; defaults to `-rc.1` |
| #1118 | RC smoke tests — same health checks as `release.yml`, run against RC-tagged images; failures block promotion |
| #1121 | Documentation — admin manual and release docs updated with RC testing flow |

RC images are **never** tagged as `latest`, major, or minor. They carry only the full `-rc.N` tag until manually promoted.

## Deployment Notes

### For operators already on v1.15.0

No breaking changes. Standard upgrade path:

```bash
docker compose pull
docker compose down
docker compose up -d
```

No new environment variables are required. All infrastructure fixes (nginx thumbnail routing, RabbitMQ configuration) take effect automatically on the new images.
