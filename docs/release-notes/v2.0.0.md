# Aithena v2.0.0 Release Notes ‚Äî React Admin Portal and Streamlit Removal

_Date:_ 2026-04-20
_Prepared by:_ Copilot

## Summary

Aithena v2.0.0 is a **major release** centered on the complete replacement of the Streamlit-based admin dashboard with a modern React single-page application (SPA). This is the headline change and represents several months of work across 30 commits on the `dev` branch.

### Highlights

1. **React admin portal** ‚Äî A full rewrite of the admin dashboard as a React SPA with 7 pages: Dashboard, Document Manager, Reindex Library, Indexing Status, System Status, Infrastructure, and Log Viewer. The admin UI is now served by `aithena-ui` at `/admin/`.
2. **Admin API endpoints** ‚Äî New REST API in `solr-search` for admin operations including pause/resume indexing, container management, and collection statistics.
3. **Streamlit admin removed** ‚Äî The `aithena-admin` container image is no longer built or published. The `src/admin/` directory has been deleted.
4. **Installer overhaul** ‚Äî GPU auto-detection (NVIDIA/Intel), SSL setup, dev/prod profiles, and compose override management.
5. **Solr 9/10 compatibility** ‚Äî Forward-compatible schema layer and collection export/import tooling for the upcoming Solr 10 migration.
6. **Accessibility** ‚Äî 38 axe-core WCAG 2.1 AA automated tests covering all pages.
7. **Security workflows** ‚Äî CodeQL analysis, GHAS alert triage, and PAT expiry detection.

## Breaking Changes

### 1. `aithena-admin` Docker Image Removed

The `aithena-admin` container image is **no longer built or published** to `ghcr.io`. Deployments that reference `ghcr.io/jmservera/aithena-admin` must be updated.

**Migration:** The React admin portal is now part of `aithena-ui` and served at `/admin/`. No separate container is needed.

**Impact on:**
- `docker-compose.yml` / `compose.prod.yml` ‚Äî Remove any `streamlit-admin` service definition
- Nginx configurations ‚Äî Update proxy rules from Streamlit paths to the React SPA at `/admin/`
- Monitoring/health checks ‚Äî Replace `/admin/streamlit/_stcore/health` with `/admin/`

### 2. Installer Generates `start.sh` with Compose Overrides

The installer (`setup.py`) now generates `start.sh` with compose override chains. Existing manual `docker compose` commands may need updating to match the new override structure.

**Migration:** Re-run the installer to regenerate `start.sh`, or manually update your compose commands to include the appropriate override files.

## Upgrade Path

### From v1.19.0 ‚Üí v2.0.0

1. **Back up your data:**
   ```bash
   # Export Solr collections (new tooling available in v2.0.0)
   docker compose exec solr solr export -collection books -out /var/solr/data/backup
   ```

2. **Pull new images:**
   ```bash
   docker compose pull
   ```

3. **Remove the Streamlit admin service** from your compose configuration:
   - Delete the `streamlit-admin` service from `docker-compose.yml` or your override files
   - Remove any Streamlit-specific nginx proxy rules

4. **Re-run the installer** (recommended):
   ```bash
   python3 setup.py
   ```
   This regenerates `start.sh` with the new compose override chain.

5. **Restart all services:**
   ```bash
   docker compose up -d
   ```

6. **Verify the admin portal:**
   ```bash
   curl -sf http://localhost/admin/ && echo " ‚úÖ React admin portal accessible"
   curl -sf http://localhost/v1/health && echo " ‚úÖ API healthy"
   ```

### From v1.18.x or earlier

Follow the [v1.19.0 deployment notes](v1.19.0.md#deployment-notes) first, then apply the v2.0.0 steps above.

## Deployment Notes

### Container Images

v2.0.0 publishes **5 container images** (down from 6):

| Service | Image |
|---------|-------|
| aithena-ui | `ghcr.io/jmservera/aithena-aithena-ui` |
| document-indexer | `ghcr.io/jmservera/aithena-document-indexer` |
| document-lister | `ghcr.io/jmservera/aithena-document-lister` |
| embeddings-server | `ghcr.io/jmservera/aithena-embeddings-server` |
| solr-search | `ghcr.io/jmservera/aithena-solr-search` |

The `aithena-admin` image is **no longer published**.

### New Environment Variables

No new required environment variables. The `VERSION` in `.env` should be updated to `2.0.0`.

### Configuration Changes

| Change | File | Required action |
|---|---|---|
| Remove `streamlit-admin` service | `docker-compose.yml` | **Required.** Delete the service definition. |
| Update nginx config for admin | `nginx.conf` | **Required if customized.** Point `/admin/` to `aithena-ui`. |
| Re-run installer | `setup.py` | **Recommended.** Regenerates `start.sh` with new override chain. |

### Data Migration

**None required.** The Solr collection schema, index format, and auth database are unchanged. Existing data is fully compatible.

## What's New in Detail

### React Admin Portal

The admin dashboard has been completely rewritten as a React SPA with the following pages:

| Page | PR | Description |
|------|-----|-------------|
| Dashboard | #1117 | Overview of system health, indexing status, and key metrics |
| Document Manager | #1122 | Browse, search, and manage indexed documents |
| Reindex Library | #1124 | Trigger full or partial re-indexing of the document library |
| Indexing Status | #1125 | Real-time view of indexing pipeline progress |
| System Status | #1126 | Service health, versions, and configuration |
| Infrastructure | #1128 | Container and resource management |
| Log Viewer | #1127 | Stream and search service logs |

**Key features:**
- Role-based access control with auth guard (#1116)
- 119 integration tests for hooks and components (#1131)
- 38 axe-core WCAG 2.1 AA accessibility tests (#1475)
- Client-side routing ‚Äî all admin pages are served from `/admin/`

### Admin API

New REST endpoints in `solr-search` power the admin portal:

- `POST /v1/admin/indexing/pause` / `POST /v1/admin/indexing/resume` ‚Äî Pause and resume the document indexer (#1232)
- `GET /v1/admin/containers` ‚Äî Container management and status
- `GET /v1/admin/collections/stats` ‚Äî Solr collection statistics (#1115)

### Solr 9/10 Compatibility

- Forward-compatible schema layer (#1365) ‚Äî Prepares for Solr 10 migration without breaking Solr 9
- Collection export/import tooling (#1363, #1456) ‚Äî Backup and restore utilities for safe migration

### Installer Overhaul (#1448)

- GPU auto-detection for NVIDIA and Intel GPUs
- SSL setup integration
- Dev/prod profile selection
- Compose override chain management via generated `start.sh`

### CI/CD Improvements

- **Workflow consolidation** (#1449) ‚Äî 29 ‚Üí 26 workflows, ‚àí154 lines
- **CodeQL analysis** (#1129) ‚Äî Explicit workflow replacing broken default setup
- **GHAS triage** (#1431) ‚Äî Automated code scanning alert triage
- **PAT expiry detection** (#1453) ‚Äî Automated notification across workflows

## Known Issues

### 1. Installer Override Chain

The new installer generates `start.sh` with compose override chains. Users who have customized their `docker compose` invocations may need to update them. The installer will warn if it detects an existing `start.sh`.

### 2. Solr 10 Migration Not Yet Complete

The Solr 9/10 compatibility layer is included but the actual Solr 10 migration is planned for a future release. See [Solr 10 Migration Plan](../prd/solr-10-migration.md) for details.
