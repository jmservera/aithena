# Aithena v1.8.0 Release Notes — UI/UX Improvements

_Date:_ 2026-03-19  
_Prepared by:_ Newt (Product Manager)

Aithena v1.8.0 delivers the **UI/UX Improvements** milestone, focusing on design system implementation, modern component patterns, mobile responsiveness, and accessibility compliance. This release transforms the user interface with a cohesive design language, improved loading and error states, and full mobile device support.

## Summary of shipped changes

### Design System & Components
- **Design tokens (CSS custom properties)** — Introduced a centralized design system using CSS custom properties for colors, typography, spacing, and shadows, enabling consistent theming across all components and simplifying future design updates (#510)
- **Lucide React icon library adoption** — Replaced emoji with professional, scalable SVG icons from Lucide React, improving visual consistency and accessibility (#511)
- **Loading states and skeleton screens** — Added intelligent loading indicators with skeleton screens for data-fetching operations, reducing perceived latency and improving user feedback (#508)

### User Experience
- **Mobile-responsive design** — Implemented responsive layout with mobile-first breakpoints for phones, tablets, and desktops, enabling full functionality on all device sizes (#509)
- **Improved empty and error states** — Enhanced messaging and visual design for empty searches, failed requests, and error scenarios to guide users toward resolution (#513)

### Backend Enhancements
- **Search rate limiting** — Implemented Redis-based rate limiting on the solr-search `/v1/search` endpoint (50 requests per 15 minutes per IP) to prevent abuse while maintaining normal user workflows (#516)

### Infrastructure & Quality
- **Pre-release integration test process** — Established Docker Compose integration testing framework for validating multi-service deployments before release (#542)
- **Screenshot automation workflow** — Created Playwright-based screenshot capture and GitHub Actions workflow for automated UI documentation (#530, #531, #532)
- **Release documentation automation** — Implemented GitHub Actions integration with Copilot CLI for automated release notes generation (#523)
- **Repository settings** — Enabled GitHub Actions to create PRs for automated workflows (#534)

### Bug Fixes
- **UI version display** — Fixed UI header to correctly display the running application version instead of a stale cached version (#545)
- **solr-search auth directory permissions** — Corrected permissions issue on the auth storage directory when using host bind mounts (#543)

## Accessibility improvements

This release significantly improves WCAG 2.1 AA compliance:

- **Icon accessibility:** Lucide React icons include proper semantic HTML and ARIA labels, improving screen reader support
- **Loading states:** Skeleton screens and loading indicators provide visual and textual feedback to assistive technologies
- **Responsive design:** Mobile layouts maintain full keyboard navigation and screen reader support on all device sizes
- **Error messaging:** Enhanced error and empty state messages are clearly visible and properly announced to assistive technologies

All search, library, upload, and admin features are fully functional on mobile devices.

## Upgrade instructions

### For users running Aithena v1.7.x or later

1. Update to the v1.8.0 release:
   ```bash
   git fetch origin
   git checkout v1.8.0
   ```

2. Verify the new design tokens and icons load correctly:
   - Search for a book and confirm icons render (Lucide SVG)
   - Check that skeleton screens appear while loading results
   - Verify empty and error states display helpful messages

3. Test mobile responsiveness:
   - Open the application on a mobile device or browser DevTools mobile emulation
   - Confirm all features are accessible and functional

4. No new environment variables, configuration changes, or database migrations are required.

### For Docker Compose deployments

If deploying via Docker Compose:

```bash
docker compose pull
docker compose up -d
```

The new design tokens and rate limiting are automatically active. Rate limiting applies only to the `/v1/search` endpoint and does not require configuration.

### Rate limiting details

- **Endpoint:** `POST /v1/search`
- **Limit:** 50 requests per 15 minutes per IP address
- **Response:** `429 Too Many Requests` when exceeded
- **Use case:** Prevents abuse from automated scrapers while maintaining normal user workflow (typical searches complete in <1 second)

If you need to adjust the rate limit for your deployment, set the following environment variables in the solr-search service:

```env
RATE_LIMIT_REQUESTS=50          # Requests per window
RATE_LIMIT_WINDOW_MINUTES=15    # Time window in minutes
```
