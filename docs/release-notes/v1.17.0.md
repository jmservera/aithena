# Aithena v1.17.0 Release Notes — GPU Acceleration & Security Updates

_Date:_ 2026-03-27  
_Prepared by:_ Newt (DevRel / Technical Writer)

## Summary

Aithena v1.17.0 is a **GPU acceleration release** introducing optional NVIDIA and Intel GPU support for document indexing (2–4× speedup on NVIDIA, 1.5–2× on Intel). GPU support is completely opt-in; CPU-only deployments are unaffected. The release also includes security vulnerability updates for `requests` and `picomatch`, embeddings-server build optimization, and comprehensive GPU documentation.

## Verified Scope for v1.17.0

### GPU Acceleration (WI-4 through WI-12)

A complete GPU acceleration feature set for document embeddings indexing:

#### WI-4: Embeddings-Server GPU Support (#1151)

**What changed:** Updated embeddings-server Dockerfile to include OpenVINO library (`openvino==2024.1.0`) for Intel GPU/CPU support alongside existing NVIDIA CUDA support.

**User impact:** GPU support now available; CPU mode unchanged by default.

#### WI-5: Device & Backend Selection (#1152)

**What changed:** Embeddings-server inference now accepts two environment variables:

- `DEVICE`: auto (default), cpu, cuda, xpu — controls which accelerator to use
- `BACKEND`: torch (default) or openvino — selects the inference engine

**Auto-fallback logic:** If `DEVICE=auto` and no GPU is detected, automatically falls back to CPU without silent failure.

**Logging:** Device selection and fallback events are logged at INFO/WARNING levels for debugging.

**User impact:** Users can opt-in to GPU acceleration with environment variables; CPU mode is unchanged.

#### WI-6: Docker Compose GPU Profiles (#1153)

**What changed:** Updated `docker-compose.yml` to support GPU device passthrough via profiles:

- `docker compose --profile nvidia up` — passes NVIDIA GPU with nvidia runtime
- `docker compose --profile intel up` — passes Intel GPU via `/dev/dri` device binding
- `docker compose up` (default) — CPU-only mode unchanged

**User impact:** Operators can enable GPU support by selecting a profile at startup with no manual configuration.

#### WI-7: Docker Compose NVIDIA Override Example (#1154)

**What changed:** New documentation file `docs/examples/docker-compose.nvidia.override.yml` provides a standalone example for NVIDIA GPU users to merge with the main compose file.

**User impact:** Clear, copy-paste-ready examples for NVIDIA GPU setup.

#### WI-10: User Manual GPU Documentation (#1157)

**What changed:** Updated `docs/user-manual.md` with GPU acceleration section covering:

- Performance expectations (2–4× NVIDIA, 1.5–2× Intel)
- NVIDIA prerequisites and setup (drivers, NVIDIA Container Toolkit) for Ubuntu/Debian and RHEL/CentOS
- Intel GPU prerequisites and setup (Arc GPU, compute-runtime)
- Intel CPU optimization path (OpenVINO, no GPU needed)
- Environment variable reference (DEVICE, BACKEND)
- Docker Compose usage examples for each GPU scenario
- Troubleshooting section (CUDA not detected, OpenVINO fallback)

**User impact:** Users now have comprehensive, step-by-step GPU setup documentation in the user manual.

#### WI-11: Admin Manual GPU Documentation (#1158)

**What changed:** Updated `docs/admin-manual.md` with GPU prerequisites and troubleshooting sections covering:

- NVIDIA Container Toolkit installation (Ubuntu/Debian, RHEL/CentOS, Windows WSL2)
- Intel compute-runtime installation
- WSL2 GPU passthrough setup
- GPU detection and verification (log inspection, `nvidia-smi`, Intel GPU tools)
- CUDA version mismatch troubleshooting
- Driver update procedures
- GPU memory utilization monitoring
- Expected throughput metrics by hardware

**User impact:** Administrators have complete deployment guidance for GPU environments.

#### WI-12: GPU Troubleshooting Guide (#1159)

**What changed:** New documentation file `docs/guides/gpu-troubleshooting.md` providing standalone troubleshooting guidance:

- Expected log messages for each scenario (GPU found, fallback, errors)
- Common GPU issues and solutions
- Hardware-specific troubleshooting (NVIDIA, Intel)
- Performance expectations by hardware type
- Driver download links
- Verification steps with driver tools
- FAQ section
- WSL2-specific GPU issues

**User impact:** Dedicated troubleshooting resource for GPU-related problems.

### Security: Dependency Updates (#1238)

**What changed:** Merged Dependabot PRs to resolve medium-severity vulnerabilities:

| Package | Vulnerability | Fix Version | Services Updated |
|---------|---|---|---|
| `requests` | Insecure Temp File Reuse in `extract_zipped_paths()` | 2.33.0 | document-indexer (#1228), solr-search (#1229), admin (#1230) |
| `picomatch` | Method Injection in POSIX Character Classes | 4.0.4 | aithena-ui (#1219) |

**User impact:** Security vulnerabilities resolved; no functional changes.

### Embeddings-Server Build Optimization (#1231)

**What changed:** Issue filed to create a separate base image for embeddings-server to speed up CI builds by ~50% (target: <30s code-only rebuilds vs. current ~60–90s).

**Proposal:** Extract the model-downloader stage into a separate Docker base image (`ghcr.io/jmservera/embeddings-server-base:multilingual-e5-base`) to avoid re-downloading the ~1GB model on every build.

**Status:** Issue closed with implementation planned for v1.17.0 or v1.18.0.

**User impact:** Faster container builds; no user-facing changes.

## Deployment Notes

### For operators already on v1.16.0

**No breaking changes.** Standard upgrade path:

```bash
docker compose pull
docker compose down
docker compose up -d
```

CPU-only deployments continue to work without any changes. To enable GPU support, follow the GPU setup instructions in the updated Admin Manual.

### GPU Setup

To opt-in to GPU acceleration:

1. Install prerequisites (NVIDIA Container Toolkit for NVIDIA GPUs, Intel compute-runtime for Intel GPUs)
2. Set environment variables: `DEVICE=cuda` (or `xpu` for Intel) and `BACKEND=torch` (or `openvino`)
3. Use the appropriate Docker Compose profile:

```bash
# NVIDIA GPU
docker compose --profile nvidia up -d

# Intel GPU
docker compose --profile intel up -d

# CPU-only (default)
docker compose up -d
```

4. Verify GPU detection in container logs
