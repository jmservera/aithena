# Aithena v1.12.2 Release Notes — Embeddings Hotfix

_Date:_ 2026-03-24  
_Prepared by:_ Newt (Product Manager)

## Summary

Aithena v1.12.2 is a focused hotfix release for the embeddings pipeline. The release context identifies one release-specific issue, **#949**, which reports that the v1.12.1 `embeddings-server` images shipped without `model_utils.py`, causing the service to fail at startup and breaking **semantic** and **hybrid** search modes.

Keyword search remains available in the reported failure mode. This release restores the embeddings containers so semantic and hybrid search can be used again.

- ✅ **Embeddings container startup fix** — restores `embeddings-server` import path
- ✅ **Semantic search recovery** — resolves the release-blocking embeddings failure reported against v1.12.1
- ✅ **Hybrid search recovery** — restores hybrid mode once embeddings startup succeeds
- ⚠️ **Operator validation required** — no CI execution log was provided in the release context

## Shipped Changes

### Issue #949: HOTFIX — embeddings-server fails to start

**Problem:**  
The v1.12.1 release shipped broken `embeddings-server` images. On startup, the container raised:

```text
ModuleNotFoundError: No module named 'model_utils'
```

**Impact:**

- `embeddings-server` cannot start correctly
- semantic search is unavailable
- hybrid search is unavailable
- keyword search still works

**Root Cause (from issue context):**

- `model_utils.py` was added to the codebase
- the Docker image build instructions did not include that file in the image
- both the builder/runtime copy path for the embeddings image were affected

**Hotfix scope:**

- update the embeddings image so `model_utils.py` is present at runtime
- restore normal startup behavior for embeddings-backed search

**Operator verification:**

- [ ] `embeddings-server` starts without import failures
- [ ] `/health` succeeds for embeddings service
- [ ] semantic search returns results
- [ ] hybrid search returns results
- [ ] keyword search still behaves as before
