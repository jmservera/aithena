# Aithena v1.17.1 Release Notes — Security Patch

_Date:_ 2026-03-28  
_Prepared by:_ Newt (DevRel / Technical Writer)

## Summary

Aithena v1.17.1 is a **security patch release** that hardens GitHub Actions CI secret handling by moving secrets behind a dedicated environment protection rule. This release does not introduce user-facing changes or deployment modifications. It builds on v1.17.0 (GPU acceleration for embeddings indexing) and applies a targeted security fix to the CI/CD pipeline.

## Verified Scope for v1.17.1

### Security: CI Secrets Hardening (#1237)

**What changed:** GitHub Actions workflow `build-containers.yml` now uses a dedicated `build` environment with protection rules to gate access to secrets (`GHCR_TOKEN`, `GHCR_USERNAME`). Secrets are no longer referenced directly in the workflow; they are now accessible only when the workflow runs in the protected environment.

**Code scanning alert resolved:** This change resolves GitHub code scanning alert #230 (zizmor — secrets without environment protection).

**Security impact:** Reduces risk of unintended secret exposure in fork scenarios or malicious pull requests. Workflows running outside the protected environment cannot access these secrets.

**User impact:** **None.** This is a CI/CD infrastructure change; no user-facing features, deployment procedures, or functionality are affected.

**Backward compatibility:** ✅ Complete. CPU-only and GPU-enabled deployments continue to work without any changes.

## Deployment Notes

### For operators already on v1.17.0

**No breaking changes. No deployment changes required.**

Users already running v1.17.0 can upgrade using the standard Docker Compose update path:

```bash
docker compose pull
docker compose down
docker compose up -d
```

All existing configurations (CPU-only or GPU-enabled) continue to work without modification.

### CI/CD Security Note

If you have forked Aithena or operate a CI/CD system that references this project, the updated `build-containers.yml` now requires a GitHub Actions environment named `build` to be configured with appropriate protection rules. For details, see the [GitHub Actions Environments](https://docs.github.com/en/actions/deployment/targeting-different-environments/using-environments-for-deployment) documentation.
