# Release v1.15.0: Release Quality & CI Hardening

**Date:** 2026-03-24
_Prepared by:_ Newt (Product Manager)

## Summary

Aithena **v1.15.0** is a release-quality and infrastructure hardening release that delivers comprehensive CI/CD improvements, admin portal enhancements, and critical bug fixes. This release adds per-container smoke tests, parallel CI pipelines, a redesigned admin portal with sidebar navigation and log viewer, SSO passthrough for Solr, and fixes for document indexer OOM crashes and thumbnail write failures. The test suite has grown from 628 to 1,944 tests — a 3× increase reflecting the quality-first focus of this milestone.

## What's New

### CI/CD & Release Workflow — #1004, #1005, #1006, #1008, #1013, #1014, #1022

- **Per-container smoke tests** added to the release workflow for fine-grained per-service validation (#1004, #1063)
- **Parallel smoke tests + faster embeddings build** via HF_TOKEN passthrough, reducing CI time significantly (#1005, #1052)
- **Pre-release log analyzer improvements** — allowlist for known warnings, configurable severity thresholds, and error count limits to reduce false positives (#1006, #1062)
- **Stale tag prevention** — release workflow race conditions eliminated to prevent tag conflicts (#1008, #1056)
- **Main→dev sync automation** — automatic branch sync after release merges keeps dev aligned (#1013, #1057)
- **Release checklist CI gate** — documentation and validation requirements enforced as a required CI check (#1014, #1060)
- **Flaky E2E test handling** — readiness gate and retry strategy for Playwright tests (#1022, #1073)

### Admin Portal — #1069, #1070, #994

- **Sidebar navigation redesign** — organized menu with grouped admin functions for better discoverability (#1069, #1085)
- **Per-service log viewer** — stream and inspect logs for individual services directly from the admin portal (#1070, #1086)
- **Detailed indexing status** — enhanced status view with per-document progress and failure details (#1086)
- **SSO: Solr admin passthrough** — admin portal injects BasicAuth credentials for Solr admin access, eliminating separate Solr login (#994, #1081)

### Testing — #1010, #1012, #1015

- **RabbitMQ permission integration tests** covering all queue operations (bind, publish, consume, ack) (#1010, #1080)
- **E2E tests for protected endpoints** — Playwright tests for upload, stats, and admin flows (#1012, #1079)
- **Nginx proxy limit validation** — tests ensure nginx limits match backend configuration (#1015, #1068)

### Infrastructure & Build — #1007, #1009, #1011, #1072, #1078

- **Volume permission hardening** — entrypoint wrapper for document-lister ensures correct permissions (#1007, #1071)
- **Health checks for nginx and aithena-ui** — container-level health probes for better orchestration (#1009, #1055)
- **Solr stats/facet type coercion** — handles string responses from Solr with automatic numeric casting (#1011, #1059)
- **PDF font installation** — custom Solr Dockerfile installs fonts for accurate PDF text extraction (#1072, #1076)
- **Build-time dependency installation** — all runtime packages installed during image build, eliminating startup downloads (#1078, #1083)
- **4-stage Docker build for embeddings-server** — 80% faster code-change rebuilds (#1064)

### Bug Fixes — #1074, #1077, #1065

- **Document indexer OOM fix** — memory limits tuned to prevent exit code 137 on large PDFs (#1074, #1075)
- **Thumbnail write failure fix** — thumbnails redirected to writable volume path (#1077, #1084)
- **Indexing progress sync** — Redis key pattern aligned between System Status and indexer namespace (#1065, #1082)
- **Noisy PDFBox warnings suppressed** in Solr logs (#1054)
- **Log analyzer duplicate issues** prevented (#1053)
- **Sync workflow removed** — merge commits enforced for release integrity (#1061)

### Security — #994, #1066

- **Simplified internal service authentication** — streamlined auth layer reduces attack surface (#1066)
- **SSO passthrough** — admin portal users access Solr admin without separate credential management (#994, #1081)

## Breaking Changes

None. All changes are backward-compatible.

- **Admin portal navigation** has been redesigned with a sidebar, but all existing routes remain functional.
- **Solr auth passthrough** is additive — existing direct Solr access is unaffected.
- **Redis key pattern alignment** is backward-compatible; the indexer and status page now agree on namespace.

## User-Facing Improvements

- **Admin portal sidebar** — cleaner navigation with grouped admin functions
- **Per-service log viewer** — administrators can inspect service logs directly from the admin UI
- **SSO for Solr admin** — no separate Solr login needed from the admin portal
- **Indexing status detail** — more granular progress reporting in the admin dashboard
- **Faster search results** — Solr stats/facet type coercion prevents 500 errors on stats pages

## Operator Improvements

- **THUMBNAIL_DIR environment variable** — document-indexer thumbnail output redirected to writable volume
- **OOM fix** — document-indexer no longer crashes on large PDFs
- **Build-time dependencies** — no runtime downloads at container start (faster cold starts)
- **PDF fonts in Solr** — correct text extraction for documents with non-standard fonts
- **Health checks** on nginx and aithena-ui for orchestration readiness

## Security

- Simplified internal service authentication reduces credential management complexity (#1066)
- SSO passthrough for Solr admin eliminates credential sharing (#994)

## Upgrade Instructions

1. Pull the latest images:
   ```bash
   docker compose pull
   ```
2. If using custom thumbnail paths, set the `THUMBNAIL_DIR` environment variable:
   ```bash
   export THUMBNAIL_DIR=/data/thumbnails
   ```
3. Restart the stack:
   ```bash
   docker compose down
   docker compose up -d
   ```
4. Verify the admin portal loads with the new sidebar navigation at `/admin`
5. No database migrations required.
6. No Redis or Solr schema changes required.
