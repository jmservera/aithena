## Verified Scope for v1.15.0

The clearest `v1.15.x` operator-facing and user-facing workflows verified in the current tree are:

### Admin Portal Enhancements (v1.15.0+)

- **Sidebar navigation redesign** in the React admin UI with organized menu structure
- **Per-service log viewer** for inspecting container logs directly from the admin portal
- **Detailed indexing status** display with progress reporting
- **Solr SSO passthrough** — admin portal injects credentials to Solr admin, eliminating separate login

### Infrastructure & Build Improvements

- **Build-time dependency installation** — services install all runtime packages during image build
- **Health checks** for nginx and aithena-ui containers
- **Volume permission hardening** for document-lister service
- **Document indexer OOM hardening** for large PDF handling
- **Thumbnail write fix** — thumbnails redirected to writable volume path via `THUMBNAIL_DIR` environment variable

### What stayed stable

No user-facing regressions or breaking changes can be inferred from the supplied changelog, PR export, or CI export. The existing end-user workflows already documented in the user manual remain the current source of truth for:

- login and account management
- search modes and diacritic-insensitive search
- PDF viewing and download actions
- collections and book-detail workflows

## Deployment Notes

### For operators already on v1.14.1

No explicit migration procedure could be derived from the supplied release exports. The safest interpretation of the current evidence is:

- keep existing `v1.14.1` deployment practices
- review the updated admin manual sections for the new environment variables
- run your normal validation checklist before promoting `v1.15.0`

### For operators using admin workflows

Review the updated admin manual and user manual for:

1. **Admin portal sidebar** — new navigation structure in `/admin`
2. **Log viewer** — per-service log inspection from the UI
3. **Thumbnail storage** — configure `THUMBNAIL_DIR` if using custom paths
4. **Document indexer OOM handling** — no special action needed; OOM crashes on large PDFs are now handled

These features are operationally significant because the admin portal is the primary operator interface for service inspection and management.
