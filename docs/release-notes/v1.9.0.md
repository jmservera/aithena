# Release Notes — v1.9.0

**Release date:** 2026-03-19
**Milestone:** [v1.9.0](https://github.com/jmservera/aithena/milestone/23?closed=1)

## Summary

Major security and user management release. Introduces a complete user management system with CRUD API, admin UI, password policies, RBAC middleware, and multiple security fixes for production readiness.

## Added

- **User management API** — full CRUD endpoints for user registration, listing, updating, and deletion via `/v1/auth/` (#549, #572)
- **User management UI** — admin-only page at `/admin/users` for managing users, plus user profile page and change password form (#554, #555, #556, #579, #589)
- **Password policy enforcement** — configurable strength requirements: minimum 10 characters, at least 3 complexity categories (uppercase, lowercase, digits, special characters), no username in password (#552, #574, #584)
- **Role-based access control (RBAC)** — middleware enforcing admin/user/viewer roles across all endpoints, with comprehensive parametrized test suite (#553, #559)
- **Auth DB migrations** — versioned schema migration framework with backup/restore tooling (#557, #571)
- **Default admin seeding** — automatic admin user creation on first startup (#550)
- **Change password endpoint** — `PUT /v1/auth/change-password` for authenticated users (#551)
- **Password reset CLI** — `reset_password.py` tool for admin operations (#547)
- **Auth integration tests** — full authentication flow validation (#558, #575)
- **Stress testing PRD** — product requirements document for load testing and hardware profiling (#590)

## Fixed

- **PDF viewer broken** — added `X-Frame-Options SAMEORIGIN` to `/documents/` nginx location to allow same-origin iframe embedding (#577)
- **Embeddings server startup requests** — enforced offline mode (`HF_HUB_OFFLINE=1`, `TRANSFORMERS_OFFLINE=1`) to prevent outbound HuggingFace Hub requests (#578)
- **Login rate limiter ineffective** — rate limiter now reads real client IP from `X-Forwarded-For` (set by nginx as `$remote_addr`) instead of always seeing the proxy IP (#583)
- **Password policy not enforced** — wired up the strong `password_policy.py` in all code paths including the CLI reset tool; removed the weaker 8-char minimum from `auth.py` (#584)

## Security

- Security review of user management module (#560)
- Rate limiter now prevents IP spoofing by using `$remote_addr` instead of `$proxy_add_x_forwarded_for`
- Password policy enforced consistently across all entry points (API, CLI, admin UI)

## Upgrade Notes

- **Breaking:** Minimum password length increased from 8 to 10 characters, with 3+ complexity categories required. Existing users with weak passwords will need to update their passwords.
- The admin user is automatically seeded on first startup if no users exist.
- New database migrations run automatically on startup.
