# Aithena v1.11.0 Release Notes — Search Results Redesign & PDF Enhancements

_Date:_ 2026-03-22  
_Prepared by:_ Ripley (Lead)

## Summary

Aithena v1.11.0 delivers improvements to the search-to-document workflow through **chunking refinements**, **PDF viewer tooling**, **a new book detail modal**, and **thumbnail generation**. The release closed 27 milestone issues via 18 merged PRs.

### Key Deliverables

- ✅ **Chunking Updates** — Sentence-boundary awareness, defaults changed to 90/10 (from 400/50)
- ✅ **Chunk Text Preview** — Chunk text fields exposed in search responses
- ✅ **PDF Viewer Toolbar** — Fullscreen, download, and open-in-new-window buttons
- ✅ **Book Detail View** — Modal overlay with metadata and similar books
- ✅ **BookCard Navigation** — Clicking a book card opens the detail view
- ✅ **Metadata Editing Integration** — Editing now accessible from book detail view
- ✅ **First-Page Thumbnails** — Thumbnail generation during indexing, stored as `.thumb.jpg`
- ✅ **Nginx Thumbnail Serving** — Static thumbnail route at `/thumbnails/...`
- ✅ **Infrastructure Fix** — Removed stale `/admin/streamlit` proxy configuration

- **Sentence-boundary chunking** — Document-indexer now splits chunks at sentence boundaries (PR #837)

## Breaking Changes

### ⚠️ Chunking Configuration Changes (Requires Reindex)

v1.11.0 changes default document chunking parameters:

| Parameter | v1.10.0 | v1.11.0 | Impact |
|-----------|---------|---------|--------|
| `CHUNK_SIZE` | 400 | 90 | Smaller chunks |
| `CHUNK_OVERLAP` | 50 | 10 | Less redundancy |

**Migration Required:**

Existing documents indexed with v1.10.0 parameters must be **fully reindexed** for chunk text fields to reflect new chunking. Without reindexing, old documents will have chunks based on 400/50 parameters; new documents will use 90/10.

See "Upgrade Instructions" below for reindexing procedure.

## Upgrade Instructions

### Standard Upgrade (No Reindexing)

If you do not require chunk text fields in search responses:

```bash
git fetch origin
git checkout v1.11.0
docker compose up -d
```

Result: PDF toolbar, book detail view, and thumbnails work immediately. Chunk text fields will not be populated for existing documents.

### Full Migration (With Reindexing)

For complete v1.11.0 functionality including chunk text with new parameters:

```bash
# 1. Deploy v1.11.0
git fetch origin
git checkout v1.11.0
docker compose pull
docker compose up -d

# 2. Wait for services to stabilize
sleep 60 && docker compose ps

# 3. Clear existing index and trigger reindex
# (Specific commands depend on your indexing pipeline configuration)

# 4. Monitor reindexing progress via service logs
docker compose logs -f document-indexer
```

**Note:** Reindexing steps depend on your deployment configuration. Consult your indexing pipeline documentation for the correct procedure.
