## Summary

Aithena v2.1.0 is a **minor release** focused on deployment flexibility and resource optimization:

1. **Configurable search architecture** — New `SEARCH_ARCHITECTURE=hnsw|hybrid-rerank` environment variable. HNSW mode (default) maintains current behavior with full kNN vector search. Hybrid-rerank mode eliminates HNSW index RAM (9-28GB savings) using BM25 pre-filter + numpy cosine reranking.
2. **Configurable vector quantization** — New `VECTOR_QUANTIZATION=none|fp16|int8` environment variable reduces HNSW RAM from ~28GB (float32) to ~14GB (fp16) or ~9GB (int8) for 9M vectors with <1-3% quality loss.
3. **Single-node Docker Compose topology** — New `docker/compose.single-node.yml` overlay replaces 3 ZK + 3 Solr (~12GB) with 1 ZK + 1 Solr (~4GB). All features including RBAC and auth remain functional.
4. **Single-user mode (`AUTH_ENABLED=false`)** — Simplifies local development and CI by disabling authentication entirely.
5. **Installer topology selection** — New `--topology single-node|distributed` and `--no-auth` flags streamline initial deployment.
6. **CI/CD improvements** — New dev integration test workflow (single-node, <20 min). Release CI validates both topologies and both auth modes with matrix strategy.
7. **Deployment topologies documentation** — New `docs/deployment-topologies.md` explains resource requirements and migration paths.

**No breaking changes** for operators keeping default settings (`SEARCH_ARCHITECTURE=hnsw`, `VECTOR_QUANTIZATION=none`, `AUTH_ENABLED=true`).

**Key risks:**
- Multiple new environment variables with interdependent behavior (search architecture, quantization, auth mode, topology)
- Switching configurations requires reindexing (search architecture, quantization)
- Hybrid-rerank mode disables features (Semantic search, Similar Books)
- CI run data is absent (empty `[]` — no automated test results available)
- 200 closed issues in milestone backlog (comprehensive feature set but broad testing surface)

**Risk assessment:** The feature set is solid and well-designed for deployment flexibility. However, **absent CI data** and the large configuration matrix (4 search architectures × 3 quantization modes × 2 auth modes × 2 topologies = 48 permutations) require explicit staging validation across critical paths before production promotion.
