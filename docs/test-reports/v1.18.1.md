## Repository Spot-Checks

### 1. Solr Auth Role Assignment Fix (#1332, PR #1333)

**Expected behavior after fix:**
- Admin user retains all 4 Solr 9.7 built-in roles: `superadmin`, `admin`, `search`, `index`
- Security API operations (add/remove users, edit permissions) succeed
- Document indexing succeeds (requires `index` role)
- Collection admin operations succeed (require `admin` role)
- Readonly user assigned `search` role instead of custom `readonly`

**Where to verify in the repository:**
- `docker-compose.yml` lines ~133-134: `SOLR_AUTH_ADMIN_PASSWORD` set, no `set-user-role` call for admin
- `docker/compose.prod.yml` lines ~95-96: same pattern
- `src/solr/security.json`: 4-tier role hierarchy (`superadmin`, `admin`, `search`, `index`)
- `src/solr-search/tests/test_solr_init_script.py`: test assertions verify new role assignments

**Unit test evidence:**
- PR #1333 states "Updated tests to verify new behavior"
- Test file includes updated assertions matching Solr 9.7 defaults

**Risk if unfixed:**  
**HIGH** — Admin users cannot perform security operations, breaking user management and permission configuration. Document updates may fail due to missing `index` role.

### 2. Installer CWD Independence Fix (#1330, PR #1331)

**Expected behavior after fix:**
- `uv run installer/setup.py --help` succeeds when run from repository root
- `uv run installer/setup.py --help` succeeds when run from `installer/` directory (existing behavior preserved)
- No `ModuleNotFoundError: No module named 'aithena_common'` in either case

**Where to verify in the repository:**
- `installer/setup.py` lines ~1-4: PEP 723 inline script metadata block:
  ```python
  # /// script
  # dependencies = ["aithena-common"]
  # ///
  ```

**Unit test evidence:**
- PR #1331 states "6/6 tests pass"
- Manual validation confirms both CWD scenarios:
  - ✅ `cd /repo && uv run installer/setup.py --help`
  - ✅ `cd /repo/installer && uv run setup.py --help`

**Risk if unfixed:**  
**MEDIUM** — Installer fails when run from repo root, confusing users who follow documentation expecting CWD-independent behavior. Workaround exists (cd into `installer/` first).

## Recommended Validation Before Production Use

### Solr Auth Tests (Priority: HIGH)

1. **Verify admin role retention:**
   ```bash
   docker compose exec solr curl -u admin:admin \
     "http://localhost:8983/solr/admin/authentication" | jq '.authentication.credentials'
   ```
   Expected: Admin user has `["superadmin", "admin", "search", "index"]`

2. **Test security API operations:**
   - Add a new user via admin portal
   - Assign roles to the new user
   - Remove the user
   - Verify no permission errors

3. **Test document indexing:**
   - Upload a document via the UI
   - Verify it indexes successfully
   - Search for the document to confirm

4. **Test collection admin:**
   - View collection status in Solr admin UI
   - Verify admin can read collection configuration
   - (Optional) Create a test collection if safe

### Installer Tests (Priority: MEDIUM)

1. **Run installer from repo root:**
   ```bash
   cd /path/to/aithena-repo
   uv run installer/setup.py --help
   ```
   Expected: Help text displays, no `ModuleNotFoundError`

2. **Run full installer workflow:**
   ```bash
   uv run installer/setup.py
   ```
   Follow prompts to create `.env` and initialize volumes. Verify no import errors.

3. **Run installer from installer/ directory:**
   ```bash
   cd installer
   uv run setup.py --help
   ```
   Expected: Same behavior as before (regression check)

### Regression Tests (Priority: MEDIUM)

1. **Search sanity check:**
   - Perform a simple search in the UI
   - Verify results display correctly

2. **Admin portal sanity check:**
   - Log into admin portal
   - View service logs
   - Verify no authentication errors

3. **Document upload sanity check:**
   - Upload a new document
   - Wait for indexing to complete
   - Search for and view the document
