## Seven Features & Fixes in v1.9.0

Per release prep PR #613 and merged pull requests:

### Authentication & User Management (PRs #549, #572, #554–#556, #579, #589)

- **User Management API:** Complete CRUD endpoints for user lifecycle management
  - `POST /v1/auth/register` — user registration
  - `GET /v1/auth/users` — list all users (admin-only)
  - `PUT /v1/auth/users/{id}` — update user profile or role
  - `DELETE /v1/auth/users/{id}` — delete user account
- **User Management UI:** React admin page at `/admin/users` for user CRUD, plus self-service profile page at `/profile`
- **Default Admin Seeding:** Automatic creation of `admin` user on first startup if no users exist
- All endpoints include comprehensive integration tests and RBAC enforcement

### Security Hardening (PRs #550, #551, #552, #553, #559, #560, #577, #578, #583, #584)

1. **#553 — RBAC Middleware Foundation**
   - Enforces three-role system: `admin`, `user`, `viewer`
   - Middleware validates role on every protected endpoint
   - Prevents privilege escalation

2. **#559 — Parametrized RBAC Tests**
   - 44 parametrized test cases covering all endpoint × role × auth-state combinations
   - Validates correct 401/403/200 responses for each role
   - Covers admin/user/viewer and unauthenticated access

3. **#550 — Default Admin Seeding**
   - First startup automatically creates `admin` user if database is empty
   - Simplifies initial deployment without manual setup

4. **#551 — Change Password Endpoint**
   - `PUT /v1/auth/change-password` for authenticated users
   - Enforces strong password policy on password updates

5. **#552, #574, #584 — Password Policy Enforcement**
   - Minimum 10 characters (increased from 8)
   - At least 3 of 4 complexity categories (uppercase, lowercase, digits, special)
   - Username must not appear in password
   - Enforced in API, CLI reset tool, and admin UI
   - PR #584 fixed CLI tool to validate before hashing

6. **#560 — Security Review**
   - Comprehensive security audit of user management module completed
   - All findings addressed before merge

7. **#577 — PDF Viewer Fix**
   - Added `X-Frame-Options SAMEORIGIN` to `/documents/` nginx location
   - Allows same-origin iframe embedding for PDF viewer
   - Preserves `nosniff` protection at location level

8. **#578 — Embeddings Server Offline Mode**
   - Enforced `HF_HUB_OFFLINE=1` and `TRANSFORMERS_OFFLINE=1` environment variables
   - Prevents unintended HuggingFace Hub requests at startup
   - Uses pre-downloaded models only

9. **#583 — Login Rate Limiter Fix**
   - Fixed rate limiter to read real client IP from `X-Forwarded-For` header
   - Previously always saw nginx proxy IP (`172.x.x.x`), making rate limits ineffective
   - Prevents IP spoofing and enables proper per-user rate limiting

10. **#607 — Comprehensive RBAC Test Suite**
    - 44 parametrized test cases replacing basic tests
    - Covers all protected endpoints with admin, user, viewer, and unauthenticated roles
    - Validates correct response codes (401 vs 403)

### Infrastructure & Testing
