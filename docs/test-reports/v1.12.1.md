## Shipped Work

### Bug Fixes & UX Refinements

#### Issue #897: Collections API — Remove Mock Data Fallback

**Changes:**
- Removed `VITE_COLLECTIONS_API === "real"` environment variable check from `collectionsApi.ts`
- Collections API now calls real backend endpoints by default
- No environment variable configuration required for production deployment

**Test Coverage:**
- Create collection end-to-end test
- Add single book to collection test
- Bulk add multiple books test
- Edit collection items and notes
- Delete collection test
- Persistent data verification (reload page, verify data remains)
- Backend SQLite storage validation

---

#### Issue #898: Login Form — Add "Remember Me" Checkbox

**Changes:**
- Added checkbox input to `LoginPage.tsx`
- Updated `AuthContext.login()` signature to accept `rememberMe?: boolean` parameter
- Login POST request now includes `remember_me` field
- Session behavior:
  - **Checked:** Persistent cookie with 30-day Max-Age
  - **Unchecked (default):** Session-only cookie

**Test Coverage:**
- Checkbox visibility and labeling on login page
- Form submission with `remember_me: true` in request body
- Cookie Max-Age validation (persistent vs. session)
- Session persistence across browser restart (when checked)
- Session termination on browser close (when unchecked)
- Logout behavior (clears session regardless of setting)

---

#### Issue #896: Search Results — Truncate Text Previews

**Changes:**
- Implemented truncation logic in `BookCard.tsx`
- Chunk text limited to approximately 15-20 characters
- For keyword/hybrid searches: truncation centers around highlighted `<em>` term
- For semantic search: truncation includes ellipsis
- `BookDetailView.tsx` unchanged — continues displaying full chunk text

**Test Coverage:**
- Truncation length validation (~15-20 characters)
- Highlighted term remains visible in truncated text
- BookDetailView shows full chunk text (no truncation)
- Keyword search highlight preservation
- Semantic search truncation with ellipsis
- UI layout stability (no overflow)

---

#### Issue #894: Container Build — Add Missing libstdc++ Library

**Changes:**
- Updated `src/document-indexer/Dockerfile` runtime stage
- Added `apk add --no-cache libstdc++ libgomp libgcc`
- PyMuPDF now loads successfully at container startup
- Thumbnail generation works during document indexing

**Test Coverage:**
- Container build validation (no missing library errors)
- Container startup test (document-indexer service health)
- PyMuPDF import test (no runtime errors)
- Thumbnail file generation on new document indexing
- Thumbnail serving via nginx (static route validation)
