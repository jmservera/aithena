## Shipped Work

### Feature: Embedding Model Evaluation Infrastructure

#### Issue #926: Evaluate e5-base vs distiluse embeddings — benchmark and quality comparison

**Changes:**
- Parallel indexing infrastructure operational for dual-model evaluation (from v1.12.0)
- Comparison API endpoint (`/v1/search/compare`) enables side-by-side queries
- Benchmark test suite: 30 queries covering exact match, semantic, multilingual, fuzzy, and complex patterns
- Metrics collection framework: p50/p95/p99 latency, memory usage, index size
- Quality assessment procedures: relevance scoring, nDCG@10, manual spot-checks
- Evaluation decision gate: migrate to e5-base or proceed to v1.14.0 evaluation UI

**Test Coverage Required:**
- Stack builds and starts without errors
- Both embeddings servers initialize and respond to health checks
- Comparison API endpoint available and responsive
- Dual-collection indexing workflow completes successfully
- Benchmark queries return results from both models
- Latency metrics are collected and comparable
- Memory usage is tracked per embeddings server
- No timeout or connection errors during extended benchmarking
- Search quality can be objectively assessed using provided metrics
