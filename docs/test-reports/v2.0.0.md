## Summary

Aithena v2.0.0 is a **major release** centered on complete replacement of the Streamlit-based admin dashboard with a modern React single-page application (SPA):

1. **React admin portal** — Full rewrite as React SPA with 7 pages: Dashboard, Document Manager, Reindex Library, Indexing Status, System Status, Infrastructure, and Log Viewer. Served at `/admin/` by `aithena-ui`.
2. **Admin API endpoints** — New REST API in `solr-search` for admin operations: `POST /v1/admin/indexing/pause`, `POST /v1/admin/indexing/resume`, `GET /v1/admin/containers`, `GET /v1/admin/collections/stats`
3. **Streamlit admin removed** — `aithena-admin` container image no longer built or published. `src/admin/` deleted (**breaking change**).
4. **Installer overhaul** — GPU auto-detection (NVIDIA/Intel), SSL setup, dev/prod profiles, compose override management with generated `start.sh`
5. **Solr 9/10 compatibility** — Forward-compatible schema layer and collection export/import tooling
6. **Accessibility** — 38 axe-core WCAG 2.1 AA automated tests
7. **Security workflows** — CodeQL analysis, GHAS alert triage, PAT expiry detection
8. **119 integration tests** for admin hooks and components

**Breaking Changes:**
- `aithena-admin` Docker image removed — deployments referencing `ghcr.io/jmservera/aithena-admin` must be updated
- Installer generates `start.sh` with compose override chains — existing manual `docker compose` commands may need updating

The React admin portal has strong automated coverage with 119 integration tests and 38 accessibility tests. However, **no CI run data was captured** (`ci-run.json` is restricted), meaning automated integration test results were not available in the release evidence.

**Risk assessment:** The breaking change and absent CI data require explicit staging validation before production promotion, despite the strong automated test coverage.

## Recommended Validation Before Production Use

### Breaking Change Verification (Priority: HIGH)

1. **Verify Streamlit admin is removed:**
   ```bash
   # Should fail (image should not exist)
   docker pull ghcr.io/jmservera/aithena-admin:2.0.0 && echo "FAIL: image should not exist" || echo "✅ admin image correctly removed"
   
   # Verify 5 images (not 6)
   docker compose images | grep -c "ghcr.io/jmservera/aithena" | grep 5 && echo "✅ Correct image count" || echo "FAIL: Expected 5 images"
   ```

2. **Verify React admin loads:**
   ```bash
   # Should return HTML (React SPA)
   curl -sf http://localhost/admin/ && echo "✅ React admin accessible"
   
   # Verify auth guard (should redirect or return 401, not 200)
   STATUS=$(curl -s -o /dev/null -w "%{http_code}" http://localhost/admin/api/containers)
   [ "$STATUS" != "200" ] && echo "✅ Auth guard active" || echo "FAIL: Auth guard not working"
   ```

3. **Verify compose configuration updated:**
   ```bash
   # Should NOT contain streamlit-admin service
   grep -q "streamlit-admin" docker-compose.yml && echo "FAIL: streamlit-admin still in compose file" || echo "✅ streamlit-admin removed"
   
   # Should NOT contain old admin proxy
   grep -q "/admin/streamlit" nginx.conf && echo "FAIL: Old Streamlit proxy still in nginx config" || echo "✅ nginx config updated"
   ```
