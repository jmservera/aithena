## Repository Spot-Checks

The following areas were identified from the closed issues as the primary change surface for v1.16.0.

### 1. Search UI — Semantic & Hybrid Results (#1221, #1224)

**Expected behavior after fix:**

- Semantic and hybrid search results render thumbnails
- Semantic search chunk text uses the same UI container as keyword results (no "small box" regression)

**Where to verify in the repository:**

- `src/aithena-ui/src/` — search result components (chunk display, thumbnail rendering)
- Thumbnail rendering should be present for all search modes, not only keyword

**Risk if unfixed:** Users see inconsistent and degraded search result display across search modes.

### 2. Search UI — Keyword Context & Page Numbers (#1222, #1223)

**Expected behavior after fix:**

- Keyword search context text is truncated to ~40 words
- Keyword search results include page number and a link to open the PDF at the matched page

**Where to verify in the repository:**

- `src/aithena-ui/src/` — keyword result component, context text rendering
- Page link generation in keyword result display

**Risk if unfixed:** Users cannot navigate to the correct PDF page from keyword search; long context text degrades readability.

### 3. Similar Books Endpoint (#1220)

**Expected behavior after fix:**

- `GET /v1/search/similar_books` returns `200` with results instead of `422`
- kNN query targets chunk documents (not parent book docs)

**Where to verify in the repository:**

- `src/solr-search/` — similar books route / kNN query construction
- Verify kNN query targets the correct Solr collection field

**Risk if unfixed:** Similar books feature is entirely broken; users cannot discover related documents.

### 4. Admin Dashboard Pagination (#1138)

**Expected behavior after fix:**

- Admin indexing status list is paginated; does not render unbounded rows

**Where to verify in the repository:**

- Admin portal component for indexing status
- Pagination controls present and functional

**Risk if unfixed:** Admin dashboard degrades with large book libraries; potential UI performance issue.

### 5. Thumbnail nginx Routing (#1137)

**Expected behavior after fix:**

- Thumbnail requests return `image/jpeg`, not `text/html`
- nginx configuration correctly routes `/thumbnails/` paths to static files

**Where to verify in the repository:**

- nginx configuration file(s) — thumbnail location block
- `docker-compose.yml` or `docker-compose.prod.yml` — volume mounts for thumbnail directory

**Risk if unfixed:** All thumbnails fail to load across the entire UI.

### 6. RabbitMQ Deprecation Warning (#1136)

**Expected behavior after fix:**

- No `management_metrics_collection` deprecation warning in RabbitMQ container startup logs

**Where to verify in the repository:**

- RabbitMQ configuration file or environment variables in `docker-compose.yml`

**Risk if unfixed:** Low severity — only a warning; no functional impact. May obscure other log signals.

### 7. Pre-Release Container Workflow (#1118 – #1123)

**Expected behavior after fix:**

- `.github/workflows/build-containers.yml` exists and is a reusable workflow covering all 6 services
- `.github/workflows/pre-release.yml` exists with `workflow_dispatch` trigger on dev branch and auto-trigger on PRs to main
- RC auto-increment logic present in workflow files
- RC smoke tests defined and blocking on failure
- RC images tagged `-rc.N` only — never `latest`

**Where to verify in the repository:**

- `.github/workflows/build-containers.yml`
- `.github/workflows/pre-release.yml`
- `release.yml` — updated to use reusable workflow

**Risk if unfixed:** No formal RC validation gate before promotion to release; increases production regression risk.
